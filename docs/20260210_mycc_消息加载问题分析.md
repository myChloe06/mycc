# myCC 消息加载问题分析

**日期**：2026-02-10
**状态**：⚠️ 问题待解决

---

## 问题描述

用户反馈的问题：
1. **打开对话都要"加载消息"很久** - 首次打开会话时，长时间显示"加载中"
2. **发送消息后显示"网络错误"** - 消息发送失败，提示"network error"
3. **退出到历史记录再回来，消息又出现** - 说明消息实际发送成功了，但前端显示错误

---

## 根本原因分析

### 1. API 调用机制（src/adapters/official.ts:75-81）

```typescript
async *chat(params: ChatParams): AsyncIterable<SSEEvent> {
  const { message, sessionId, cwd, images, model } = params;

  // 每次都调用 SDK query()
  for await (const sdkMessage of query({...})) {
    yield sdkMessage as SSEEvent;
  }
}
```

**问题**：
- **每个消息都新建一次 API 连接**
- **没有连接复用机制**
- **message_id 为空字符串**（SDK 自动填充）

### 2. SSE 流式传输（src/http-server.ts:224-257）

```typescript
// 设置 SSE 响应头
res.writeHead(200, {
  "Content-Type": "text/event-stream",
  "Cache-Control": "no-cache",
  "Connection": "keep-alive",
});

let currentSessionId = sessionId;

try {
  // 使用 adapter 的 chat 方法（返回 AsyncIterable）
  for await (const data of adapter.chat({...})) {
    // 提取 session_id
    if (data && typeof data === "object" && "type" in data) {
      if (data.type === "system" && "session_id" in data) {
        currentSessionId = data.session_id as string;
      }
    }
    res.write(`data: ${JSON.stringify(data)}\n\n`);
  }
  // 完成
  res.write(`data: ${JSON.stringify({ type: "done", sessionId: currentSessionId })}\n\n`);
  res.end();
}
```

**问题**：
- **SSE 连接不稳定** - 容易中断
- **没有重试机制** - 连接失败后不会自动重连
- **没有心跳保活** - 长时间无消息容易超时

### 3. 日志记录（src/http-server.ts:224）

```typescript
console.log(`[CC] 收到消息: ${message.substring(0, 50)}...${hasImages ? ` (附带 ${images.length} 张图片)` : ""}`);
```

**问题**：
- **只记录收到，不记录发送结果**
- **无法追踪完整流程**

## 建议的解决方案

### 方案 1: 优化 API 调用（推荐）

**目标**：复用连接，减少延迟

**修改点**：`src/adapters/official.ts`

```typescript
// 在类中添加连接缓存
private claudeConnection: any = null;
private lastConnectionTime: number = 0;
private CONNECTION_TTL = 5 * 60 * 1000; // 5分钟

async *chat(params: ChatParams): AsyncIterable<SSEEvent> {
  const { message, sessionId, cwd, images, model } = params;

  // 检查连接是否可用（5分钟内）
  const now = Date.now();
  if (this.claudeConnection &&
      now - this.lastConnectionTime < CONNECTION_TTL) {
    console.log(`[CC] 复用现有连接（年龄：${Math.floor((now - this.lastConnectionTime) / 1000)}秒）`);
  } else {
    // 创建新连接
    console.log(`[CC] 创建新连接`);
    this.claudeConnection = await query({
      stream: true,
      ...sdkOptions
    });
    this.lastConnectionTime = now;
  }

  // 复用连接发送消息
  for await (const sdkMessage of query({
    message: [{ role: "user", content: message }],
    stream: true,
    ...sdkOptions,
  })) {
    yield sdkMessage as SSEEvent;
  }

  // 注意：不要关闭连接，保持复用
}
```

### 方案 2: 增强 SSE 稳定性

**目标**：添加心跳和重试机制

**修改点**：`src/http-server.ts`

```typescript
// 添加心跳计时器
const HEARTBEAT_INTERVAL = 30000; // 30秒

private async handleChat(req: http.IncomingMessage, res: http.ServerResponse) {
  // ... 现有代码 ...

  const heartbeatTimer = setInterval(() => {
    res.write(`: heartbeat\n\n`); // 发送心跳
  }, HEARTBEAT_INTERVAL);

  req.on('close', () => {
    clearInterval(heartbeatTimer);
    console.log(`[CC] SSE 连接关闭`);
  });
}
```

### 方案 3: 改进日志

**修改点**：`src/http-server.ts:224`

```typescript
// 收到消息
console.log(`[CC] ✅ 收到消息: ${message.substring(0, 50)}...${hasImages ? ` (附带 ${images.length} 张图片)` : ""}`);

const startTime = Date.now();

try {
  // ... 处理消息
} finally {
  const duration = Date.now() - startTime;
  console.log(`[CC] 📤 消息处理完成: ${duration}ms`);
}
```

## 优先级

| 优先级 | 方案 | 难度 | 预期效果 |
|--------|------|--------|---------|
| **高** | 方案 1：连接复用 | 中 | 大幅降低延迟 |
| **中** | 方案 2：SSE 心跳 | 中 | 提升稳定性 |
| **低** | 方案 3：改进日志 | 低 | 便于调试 |

---

## 建议

**先从方案 3 开始**（改进日志），这样可以更清楚地看到问题发生的位置和原因。

**然后实施方案 1**（连接复用），这应该能解决"加载消息"慢的问题。
